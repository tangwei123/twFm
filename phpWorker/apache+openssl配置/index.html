方法一（从头编译apache，使用https）： 1、openssl生成证书 2、编译apache支持mod_ssl.so模块，期间需要apr、apr-util、apr-iconv、pcre包的支持 3、修改httpd.conf和httpd-ssl.conf文件， 修改httpd.conf: LoadModule ssl_module modules/mod_ssl.so、 include httpd-ssl.conf、 listen 443 修改httpd-ssl.conf： 把密钥文件和证书文件加上、 建立一个SSL的虚拟站点，避免SSL配置影响原来HTTP的站点配置 DocumentRoot “C:/Program Files/Apache2/htdocs” ServerName 域名:443 SSLEngine on 整个站点，除了虚拟站点外，关闭或打开SSL，仍然支持HTTP通信 SSL功能打开，如果在Virtualhost 出现这句，则仅作用于虚拟机站点配置范围，这个虚拟机站点全部使用SSL通信，如果出现在Virtualhost外，则作用于全局，整个服务器都使用 SSL（HTTPS）通信，不能采用HTTP通信，所以通常都在Virtualhost中加这句 SSLCertificateFile /opt/apache2/conf/ssl.key/server.crt SSLCertificateKeyFile /opt/apache2/conf/ssl.key/server.key #116行 #SSLCertificateChainFile “C:/SSL/chain.cer” 证书链文件，GeoTrust证书除了EV以外，都是单根证书，都不需要配置这个参数，如果是EV证书，则需要加入EV的证书链 ErrorLog “C:/Program Files/Apache2/logs/error.log” TransferLog “C:/Program Files/Apache2/logs/access.log” 方法二（如果apache已经编译过了，那么后续怎么加模块，支持https） 1、openssl生成证书 2、进入你apache压缩包中的/module/ssl中，运行apxs -i -c -a -D HAVE_OPENSSL=1 -I /usr/include/openssl -lcrypto -lssl -ldl *.c 3、修改httpd.conf和httpd-ssl.conf文件，配置证书部分 4、重启apache即可 方法三（如果是是yum安装的）： 只要yum install mod_ssl就行
